Voice Cloning
-------------

Qwen3-TTS is great at voice cloning, but lacks tools.  GPT-SoVITS is good, but the tools are great.
I recommend using the voice segmentation tool and voice recognition tool from GPT-SoVITS with Qwen3-TTS for the best results.

If you want to do voice cloning with GPT-SoVITS, model V2ProPlus is currently the best model (Not V4).  


Using the tools from GPT-SoVITS
-------------------------------

In order to clone a voice with Qwen3-TTS, it's easiest to use the above features from the GPT-SoVITS package to automate the tasks required.
Using the voice segmentation also seems to improve the quality of the data so start with segmenting whatever
audio you have created for training.  I recommend changing a couple settings for the segmentation: 

Min_length - Change from 4000 to 10000
Minimum interval for audio cutting - Change from 300 to 500

Fill in the 'Voice Segmentation tool' with the folder or filename you wish to use in training. (ie c:\train\combined_audio.wav)
Click 'Turn On Speech Segmentation' and check for the completion message under 'Speech Segmentation Process Output Information'

After segmentation, use the 'Faster Whisper' speech recognition to automatically create the text labelling to match the audio.
Select the ASR model: Faster Whisper
Select the ASR language: EN  (If your cloning in English)
Click 'Turn on Speech Recognition' and check for the completion message under 'Speech Recognition Process output information'.

You will now have the following files:
Under C:\GPT-SoVITS-V2ProPlus-Windows\output\slicer_opt you will have the generated slices of training audio.
Under C:\GPT-SoVITS-V2ProPlus-Windows\output\asr_opt you will have the slicer_opt.list file which is the file listing the
paths and text of each audio file.

If the training data is too large for your amount of VRAM you will get a CUDA OOM error and generation will fail.
You may need to edit the training data to remove some of the audio files from slicer_opt.list and delete the excess audio files.
I personally remove any sliced clips that are 3 seconds or under.  We want the script to capture the tempo of the voice.

Training data (For 8GB VRAM) cannot be too large or you will get a CUDA OOM error. The largest set I have been successful 
in generating is 1 minute and 30 seconds long. Audio data size is 8.24MB or 8852KB - Also you can't have any audio clips 
over 20 seconds long or it will crash (For 8GB VRAM). Higher VRAM means you can do bigger data sets and use longer audio clips.
You only need 1 or 2 minutes of quality training audio for a good voice clone.

Copy the slicer_opt.list file (from C:\GPT-SoVITS-V2ProPlus-Windows\output\slicer_opt) to the C:\Qwen3-TTS-Windows root folder.
That is all we need GPT-SoVITS for when using Qwen3-TTS for the voice cloning.
You can do voice cloning in GPT-SoVITS also, I just don't find it as good as Qwen3-TTS.

See instructions further down to continue voice cloning with GPT-SoVITS


Qwen3-TTS Instructions (With the help of GPT-SoVITS) 
----------------------------------------------------

I made a script that automatically converts the slicer_opt.list (From GPT-SoVITS) to something we can use in Qwen3-TTS.
The slicer_opt.list file contains the filenames and paths of all spliced .wav audio clips, along with their corresponding 
text transcripts.  Qwen3-TTS for voice cloning only supports a single training audio file so the script will combine
multiple audio files and then generate the python file to run to do the actual cloning.  Generating the files takes a
split of a second and makes cloning with Qwen3-TTS much easier.

Open the create_voice_clone_slicer.py file for editing and change the 'text_content =' text to whatever you want your
cloned voice to say.  Save the edited file.

Now run the file with: python create_voice_clone_slicer.py

This will create the following 3 files:
combined_reference.txt
combined_reference.wav
generated_tts.py

Now to generate the cloned audio, just run the generated_tts.py python file: python generated_tts.py

That's it, in a bit, you should now see a file called output_voice_clone.wav with the generated clone output.
You can just edit the generated_tts.py model (change the text to generate) if you wish but by running
from the create_voice_clone_slicer.py it cleans up the text and adds some punctuation to help with better
quality cloning.  Without this step the audio output may not be correct.
I should also mention I have the script also generate two additional audio files at 95% and 105% of the original audio
files speed in case it doesn't quite sound right at the original speed.

Qwen 3-TTS Punctuation
----------------------

Use Punctuation Strategically: This is your most basic and powerful tool. Different punctuation marks instruct the model 
to create pauses of varying lengths.

Comma , : Creates a short pause (around 300ms), perfect for lists or separating clauses.

Semicolon ; : Creates a medium-length pause (around 500ms), ideal for related independent clauses.

Period . : Creates a full-stop pause (around 600-700ms), signaling the end of a thought.

Ellipsis … : Indicates a trailing-off thought, often resulting in a longer, more dramatic pause with a slight slowing of pace.

Colon : : Signals that an explanation or list is coming, creating a noticeable pause with a slight rise in intonation.

—   = emphasis shift (Builds tension, Feels more intentional / confident / dramatic on words after dash)
\n\n = full reset (Docs say huge impact but it does not appear to do anything - maybe training data issue?) 


Qwen3-TTS Recommended Settings
------------------------------

Try these different settings for Qwen3-TTS:
temperature: 0.5 – 0.65   ← LOWER than usual
top_p: 0.7 – 0.85
top_k: 20 – 40
repetition_penalty: 1.05

Defaults from official API:
temperature: 0.9
top_p: 1.0
top_k: 50
repetition_penalty: 1.05

For stable timing (Voice speed not changing so bad!):
temperature=0.45,
top_p=0.8,
top_k=30,
repetition_penalty=1.1

Lower temperature = more consistent speed

Better to do one/two/three sentences at a time instead of pages, then concatenate audio as the model tends to speed up the voice 
the longer the amount of text to generate gets.  Note: Using the segmentation in GPT-SoVITS appears to resolve this issue!
I am now able to do 60 seconds clips without this annoying issue popping up.  The training audio you are using can still affect it.


GPT-SoVITS Voice Cloning Continued
----------------------------------

After creating the slicer_opt.list file, select the second tab '1-GPT-SOVITS-TTS'.
All the needed fields will already be filled in from the segmentation tool.
Type a name or use the default 'xxx' to name your experiment.  This will populate in later tabs automatically.
When your want to clone another voice make sure you change this to a different name or you may corrupt you other experiment.

Scroll to the bottom of the page and click 'Turn on One-click triple action for training set formatting'
This will automatically run through all three needed extraction features.
A message will be displayed while carrying out each phases and a final completion message when done.

Next, select the second tab (On the second set of tabs below) called '1B-Fine-tuning training'.
Click 'Turn on SoVITS Training' and wait for the completion message.
Click 'Turn on GPT training' and wait for the completion message.
All the needed training is now done and you can move to the inference.

Next, select the third tab (On the second set of tabs below) called '1C-Reasoning'.
Click the dropdown marked 'List of GPT Models' to select the one of the models you trained.
Click the dropdown marked 'SoVITS Model List' to select the one of the models you trained.
Click 'Turn on TTS Inference WebUI' to open the inference tab in a new window. It takes a few seconds.

In order to clone you must supply an audio file with the same voice which you can drag and drop on the inference screen.
The audio clip must be between 3-10 seconds or you WILL get an error.
You must also include the text transcribed from the audio file in the 'Text for reference audio' field.
Also change the dropdowns for the language if your voice cloning in another language other than English.
Once you have done all that, your ready to actually clone a voice!

Type whatever you want the text to be for the cloned voice in the 'Inference text' box.
Click 'Start inference'. Results appear to the right of the button.  Click play to hear or download!

You can modify the temperature, top_k, and top_p but by default top_k and top_p are disabled (Set to 1 disables)

As I said before, I recommend using Qwen3-TTS as the quality is just better on voice cloning.
