Voice Cloning (GPT-SoVITS is only needed for better voice cloning, you can skip if you just want to use Qwen3-TTS Custom voices or Voice Designer)

Information on GPT-SoVITS V4/V2Pro/V2ProPlus (Used to make Qwen3-TTS easier to use for cloning through it's tools like the splicer, and speech recognition text labeling)

If you want to do voice cloning with GPT-SoVITS, model V2ProPlus is currently the best model.  
I know it's confusing because there are higher versions (V3,V4) but this is the best version.
In order to clone a voice with Qwen3-TTS, it's easiest to use several features from the GPT-SoVITS package to automate training
and data extraction.  Using the slicer seems to improve the quality of the data so start with slicing whatever audio
you have created for training.  I recommend changing a couple settings for the slicer: 

Min_length - Change from 4000 to 10000
Minimum interval for audio cutting - Change from 300 to 500

Fill in the 'Audio slicer input (file or folder)' with the path to the folder or file you wish to use for audio data to train.
Click 'Open Speech Slicing.'  When complete, a message will be displayed under: Speech Slicing Process Output Information

After slicing, use the 'Faster Whisper' speech recognition to automatically create the text matching the audio.
The paths are already set in the script when you use the slicer, so no need to modify those.

Select the ASR model: Faster Whisper
Select the ASR language: EN  (If your cloning in English)
Click 'Open Speech Recognition'

When complete, you will see a message under: Speech Recognition Process Output Information

You will now have the following files:
Under C:\GPT-SoVITS-V2ProPlus-Windows\output\slicer_opt you will have the generated slices of training audio.
Under C:\GPT-SoVITS-V2ProPlus-Windows\output\asr_opt you will have the slicer_opt.list file which is the file listing the
paths and text of each audio file.

If the training data is too large for your amount of VRAM you will get a CUDA OOM error and generation will fail.
You may need to edit the training data to remove some of the audio files from slicer_opt.list and delete the excess audio files.

Training data (For 8GB VRAM) cannot be too large or you will get a CUDA OOM error. The largest set I have been successful 
in generating is 1 minute and 30 seconds long. Audio data size is 8.24MB or 8852KB - Also you can't have any audio clips 
over 20 seconds long or it will crash (For 8GB VRAM). Higher VRAM means you can do bigger data sets and use longer audio clips.

Copy the slicer_opt.list file (from C:\GPT-SoVITS-V2ProPlus-Windows\output\slicer_opt) to the C:\Qwen3-TTS-Windows root folder.
That is all we need GPT-SoVITS for when using Qwen3-TTS for the voice cloning.
You can do voice cloning in GPT-SoVITS also, I just don't find it as good as Qwen3-TTS.


Qwen3-TTS Instructions

I made a script that automatically converts the slicer_opt.list to something we can use in Qwen3-TTS.
Qwen3-TTS for voice cloning only supports a single reference audio file so the script will combine multiple audio files.

Open the create_voice_clone_slicer.py file for editing and change the 'text_content =' text to whatever you want your
cloned voice to say.  Save the edited file.

Now run the file with: python create_voice_clone_slicer.py

This will create the following 3 files:
combined_reference.txt
combined_reference.wav
generated_tts.py

Now to generate the cloned audio, just run the generated_tts.py python file: python generated_tts.py

That's it, in a bit, you should now see a file called output_voice_clone.wav with the generated clone output.
You can just edit the generated_tts.py model (change the text to generate) if you wish but by running
from the create_voice_clone_slicer.py it cleans up the text and adds some punctuation to help with better
quality cloning.  Without this step the audio output may not be correct.

Qwen 3-TTS Punctuation

Use Punctuation Strategically: This is your most basic and powerful tool. Different punctuation marks instruct the model 
to create pauses of varying lengths .

Comma , : Creates a short pause (around 300ms), perfect for lists or separating clauses.

Semicolon ; : Creates a medium-length pause (around 500ms), ideal for related independent clauses.

Period . : Creates a full-stop pause (around 600-700ms), signaling the end of a thought.

Ellipsis … : Indicates a trailing-off thought, often resulting in a longer, more dramatic pause with a slight slowing of pace.

Colon : : Signals that an explanation or list is coming, creating a noticeable pause with a slight rise in intonation.

—   = emphasis shift (Builds tension, Feels more intentional / confident / dramatic on words after dash)
\n\n = full reset (Docs say huge impact but it does not appear to do anything - maybe training data issue?) 


Qwen3-TTS Recommended Settings

Try these different settings for Qwen3-TTS:
temperature: 0.5 – 0.65   ← LOWER than usual
top_p: 0.7 – 0.85
top_k: 20 – 40
repetition_penalty: 1.05

Defaults from official API:
temperature: 0.9
top_p: 1.0
top_k: 50
repetition_penalty: 1.05

For stable timing (Voice speed not changing so bad):
temperature=0.45,
top_p=0.8,
top_k=30,
repetition_penalty=1.1

Lower temperature = more consistent speed

Better to do one/two/three sentences at a time instead of pages, then concatenate audio as the model tends to speed up the voice 
the longer the amount of text to generate gets.  Using the slicer in GPT-SoVITS appears to resolve this issue!
I am now able to do 60 seconds clips without this annoying issue popping up.  The training audio you are using can affect it still.
